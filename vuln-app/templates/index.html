🎯 Vuln-App — cible de test WAF

Chaque carte attaque un endpoint volontairement vulnérable. Le badge indique si la requête a été bloquée par le WAF (403) ou si elle est passée — pratique pour comparer avant/après installation de ModSecurity, sans rien taper dans un terminal.

⚠️ Application volontairement vulnérable, laboratoire local uniquement. N'expose jamais ce port sur Internet.

🪤 XSS réfléchi GET /search?q=

Le paramètre q est renvoyé tel quel dans la page, sans échappement.


  


  
  

    
💉 Injection SQL POST /login

    
Requête SQL construite par concaténation directe des identifiants.

    

      

        
        
      

      

    

    

  


  
  

    
📂 Path traversal / LFI GET /file?name=

    
Le nom de fichier n'est pas validé avant lecture sur le disque.

    

      
      

    

    

  


  
  

    
🖥️ Injection de commande GET /ping?host=

    
Le host est injecté tel quel dans une commande shell (ping).

    

      
      

    

    

  


  
  

    
↪️ Open redirect GET /redirect?url=

    
Redirection vers une URL fournie par le client, sans liste blanche.

    

      
      

        
        
      

    

    

  


  
  

    
🌐 SSRF GET /fetch?url=

    
Le serveur va chercher lui-même l'URL fournie — accès possible à des services internes.

    

      
      

    

    

  


  
  

    
📤 Upload sans validation POST /upload

    
N'importe quel fichier est accepté et enregistré tel quel (type, extension...).

    

      
      

    

    

  


  
  

    
🧬 XXE POST /xml

    
Le parseur XML résout les entités externes, ce qui permet de lire des fichiers locaux.
